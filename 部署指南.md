# GitHub Pages 部署指南

## 步骤一：准备工作

### 1. 注册GitHub账号
- 访问 https://github.com
- 点击 Sign up 注册账号（如果已有账号则登录）

### 2. 安装Git（如果还没安装）
- 访问 https://git-scm.com/download/win
- 下载并安装Git for Windows

## 步骤二：创建GitHub仓库

### 1. 创建新仓库
- 登录GitHub后，点击右上角的 "+" 号
- 选择 "New repository"
- 填写信息：
  - Repository name: `qrcode-page` （或其他你喜欢的名字）
  - Description: 企业收款码页面
  - 选择 "Public" （公开）
  - ✅ 勾选 "Add a README file"
- 点击 "Create repository"

## 步骤三：上传文件到GitHub

### 方法A：使用Git命令行（推荐）

1. 打开命令行，进入 H5_pay 文件夹：
```bash
cd H5_pay
```

2. 初始化Git仓库：
```bash
git init
```

3. 添加所有文件：
```bash
git add .
```

4. 提交文件：
```bash
git commit -m "初始化收款码页面"
```

5. 关联远程仓库（替换成你的GitHub用户名和仓库名）：
```bash
git remote add origin https://github.com/你的用户名/qrcode-page.git
```

6. 推送到GitHub：
```bash
git branch -M main
git push -u origin main
```

### 方法B：使用GitHub网页上传（简单）

1. 进入你创建的仓库页面
2. 点击 "Add file" → "Upload files"
3. 将 H5_pay 文件夹中的所有文件拖拽到页面
4. 填写提交信息：`上传收款码页面文件`
5. 点击 "Commit changes"

## 步骤四：启用GitHub Pages

1. 在仓库页面，点击 "Settings"（设置）
2. 在左侧菜单找到 "Pages"
3. 在 "Source" 下：
   - Branch: 选择 "main"
   - Folder: 选择 "/ (root)"
4. 点击 "Save"
5. 等待1-2分钟，页面会显示访问链接

## 步骤五：获取访问链接

部署成功后，你会看到类似这样的链接：
```
https://你的用户名.github.io/qrcode-page/
```

这就是你的收款码页面地址！

## 步骤六：使用和分享

3. 客户点击链接即可看到收款码

## 常见问题

### Q: 页面显示404？
A: 等待3-5分钟，GitHub Pages需要时间部署

### Q: 图片不显示？
A: 确保图片已上传到 images 文件夹，且文件名正确

### Q: 想要自定义域名？
A: 在 Settings → Pages 中可以设置自定义域名

### Q: 访问速度慢？
A: GitHub在国内访问可能较慢，可以考虑使用Gitee Pages（国内）

## 使用Gitee Pages（国内替代方案）

如果GitHub访问慢，可以使用Gitee：

1. 访问 https://gitee.com 注册账号
2. 创建仓库（步骤类似GitHub）
3. 上传文件
4. 在仓库页面点击 "服务" → "Gitee Pages"
5. 点击 "启动" 即可获得访问链接

Gitee Pages 链接格式：
```
https://你的用户名.gitee.io/qrcode-page/
```
